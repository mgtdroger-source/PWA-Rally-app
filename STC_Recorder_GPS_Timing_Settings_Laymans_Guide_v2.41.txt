STC RECORDER — GPS TIMING SETTINGS GUIDE
Plain-language reference for future use

This section of index.html contains the main GPS timing settings:

    const GPS_CONFIG = {
        ...
    };

Most of the time, these settings should be left alone.

The safest rule is:

    Change only a setting you understand.
    Change one setting at a time.
    Keep a copy of the working index.html before making changes.

=====================================================
BASIC SETTINGS
=====================================================

startGpsOnOpen: true,

What it does:
Starts the phone's GPS automatically when the app opens.

true  = GPS starts automatically.
false = GPS waits until some other part of the app starts it.

Recommended:
Keep this set to true.

-----------------------------------------------------

useTimedWarmup: false,

What it does:
Controls whether the app uses a fixed timed GPS warm-up period before normal operation.

true  = the app waits for the programmed warm-up period.
false = no fixed timed warm-up is used.

Recommended:
Keep this set to false unless a future test specifically requires a timed warm-up.

-----------------------------------------------------

primaryMethod: 'M2',

What it does:
Selects the main GPS line-crossing calculation method.

'M2' means Method 2 is the normal result used by the app.

Recommended:
Keep this set to 'M2'.

-----------------------------------------------------

backupMethod: 'P',

What it does:
Selects the backup GPS calculation method.

'P' means Method P is available as the backup or insurance result.

Recommended:
Keep this set to 'P'.

-----------------------------------------------------

allowBackupIfBetterQuality: false,

What it does:
Controls whether Method P is allowed to replace Method 2 merely because its calculated quality appears better.

true  = Method P may replace Method 2 when judged better.
false = Method 2 remains the main result; Method P is backup insurance only.

Recommended:
Keep this set to false.

=====================================================
GPS ACCURACY SETTINGS
=====================================================

accuracyBands: {
    goodMaxM: 8,
    fairMaxM: 15
},

What it does:
Defines how the app labels GPS accuracy.

goodMaxM: 8

A GPS reading with estimated accuracy of 8 metres or better is classed as Good.

fairMaxM: 15

A GPS reading worse than 8 metres but no worse than 15 metres is classed as Fair.

Anything worse than 15 metres is treated as Poor.

Simple meaning:

0–8 m        = Good
over 8–15 m  = Fair
over 15 m    = Poor

Recommended:
Leave these values unchanged unless field testing proves a better limit.

-----------------------------------------------------

recordPoorGpsTimes: true,

What it does:
Controls whether the app still records GPS timing evidence when GPS accuracy is Poor.

true  = poor-quality GPS evidence may still be recorded for later inspection.
false = poor-quality GPS timing evidence is discarded.

Important:
Recording a poor reading does not mean it becomes the preferred official result. It simply preserves evidence for testing or audit.

Recommended:
Keep this set to true while GPS evidence remains useful for audit and testing.

-----------------------------------------------------

staleFixMs: 15000,

What it does:
Sets how old a GPS reading may be before the app considers it stale.

15000 means 15,000 milliseconds, which is 15 seconds.

A GPS fix older than this should not be trusted as a current position.

Recommended:
Keep this at 15000 unless future testing shows a need to change it.

=====================================================
TIME BUTTON SETTINGS
=====================================================

buttonHoldMs: 500,

What it does:
Sets how long the Time button confirmation hold or swipe period lasts.

500 means 500 milliseconds, or half a second.

A larger number makes the hold longer.
A smaller number makes it quicker.

Recommended:
Keep this at 500 unless the button feels too fast or too slow in real use.

-----------------------------------------------------

stcConfirmBuzzEnabled: true,

What it does:
Controls whether the phone plays a short audible buzz after an STC time has been successfully recorded and saved.

true  = play the confirmation buzz.
false = do not play the confirmation buzz.

Important:
The buzz happens only after the time has been accepted and saved.

It does NOT play:
- on the first touch;
- during the hold or confirm action;
- during the finger practice test;
- if the recording fails.

Normal sequence:

    confirm action
    record and save the STC time
    play the short buzz
    move to the next STC

Recommended:
Keep this set to true if an audible confirmation is useful to the driver or navigator.

-----------------------------------------------------

stcConfirmBuzzMs: 200,

What it does:
Sets the length of the audible confirmation buzz.

200 means 200 milliseconds, which is one fifth of a second.

A larger number makes the buzz longer.
A smaller number makes it shorter.

Examples:

100 = very short buzz
200 = normal short confirmation buzz
300 = slightly longer buzz

Recommended:
Keep this at 200 unless testing on the actual phone shows that a shorter or longer buzz is easier to hear.

Important:
The phone's media volume must be high enough to hear the buzz. Silent settings, Bluetooth audio or phone-specific browser behaviour may affect it.

-----------------------------------------------------

buttonModeShowGpsTrialDiagnostics: false,

What it does:
Controls whether the GPS testing and diagnostic information is visible on the screen while using Button mode.

false = clean normal-use screen.
true  = GPS testing details are shown.

Important:
This setting only hides or shows the test display.

It does NOT:
- turn GPS off;
- stop GPS evidence being recorded;
- change Method 2 or Method P;
- change the 300 metre detection gate;
- change the Time button;
- affect button-recorded times.

Normal rally use:
    buttonModeShowGpsTrialDiagnostics: false,

GPS testing:
    buttonModeShowGpsTrialDiagnostics: true,

-----------------------------------------------------

buttonModeSubmitGpsEvidence: true,

What it does:
Controls whether Button mode includes the background GPS evidence in the saved or submitted record.

true  = button time is recorded together with available GPS audit evidence.
false = GPS evidence is not included with the Button-mode submission.

Important:
The button-recorded time remains the user's recorded time. GPS is supporting audit evidence only.

Recommended:
Keep this set to true.

=====================================================
300 METRE GPS DETECTION GATE
=====================================================

gpsDetectionGateM: 300,

What it does:
Prevents the GPS line-crossing engine from reacting to a trip line too early.

The app first waits until the vehicle comes within 300 metres of the waypoint or STC centre.

Before entering the 300 metre area:
- possible crossings of the extended trip line are ignored.

After entering the 300 metre area:
- GPS line detection is armed;
- Method 2 and Method P may detect the proper crossing.

Why it is needed:
On a twisting road, the vehicle may cross the imaginary extension of a trip line while still on an earlier part of the route. The 300 metre gate helps prevent that false early reading.

Important:
This setting has no effect on the Time button.

The driver may still press the Time button normally whether the GPS gate is armed or not.

Changing the value:

300 = line detection arms within 300 metres of the point centre.

A smaller value:
- arms later;
- reduces early-crossing risk;
- may leave less time for GPS detection.

A larger value:
- arms earlier;
- gives more approach time;
- may allow false readings on nearby twisting roads.

Recommended:
Keep this at 300 metres unless field testing shows a clear reason to change it.

=====================================================
GPS MODE DISPLAY SETTINGS
=====================================================

gpsModeShowAccuracy: true,

What it does:
Controls whether GPS accuracy information is shown when the app is operating in GPS mode.

true  = accuracy is visible.
false = accuracy is hidden.

Recommended:
Keep this set to true for GPS mode testing and supervision.

-----------------------------------------------------

gpsModeTimesEditable: false,

What it does:
Controls whether GPS-generated times may be manually edited in GPS mode.

true  = the displayed GPS times can be edited.
false = GPS-generated times are locked.

Recommended:
Keep this set to false so recorded GPS results cannot be accidentally altered.

=====================================================
METHOD P SETTINGS
=====================================================

Method P is the backup or insurance calculation.

Method 2 remains the main GPS result.

These settings control how Method P checks a group of GPS readings around the crossing. They should normally not be changed individually without careful testing.

-----------------------------------------------------

methodPEvidenceWindowM: 60,

What it does:
Sets the normal distance window used by Method P when examining GPS evidence around the line.

60 means a 60 metre evidence window.

Recommended:
Keep at 60.

-----------------------------------------------------

methodPFallbackWindowM: 90,

What it does:
Sets a wider backup evidence window for Method P when the normal 60 metre window is not sufficient.

90 means a 90 metre fallback window.

Recommended:
Keep at 90.

-----------------------------------------------------

methodPDeparturePaddingM: 60,

What it does:
Keeps additional signed-distance history after the fitting area so Method P can prove that the vehicle has moved away from the line after crossing it.

60 means an extra 60 metres of departure evidence may be retained.

Recommended:
Keep at 60.

-----------------------------------------------------

methodPWindowPaddingM: 5,

What it does:
Adds a small tolerance around the Method P evidence window.

5 means 5 metres of extra padding.

This helps avoid rejecting useful readings that sit just outside the exact window boundary.

Recommended:
Keep at 5.

-----------------------------------------------------

methodPMinFitPoints: 2,

What it does:
Sets the minimum number of suitable GPS readings Method P needs to attempt a line-crossing calculation.

2 means Method P needs at least two usable points.

Recommended:
Keep at 2.

-----------------------------------------------------

methodPPreferredFitPoints: 3,

What it does:
Sets the preferred number of suitable GPS readings for Method P.

3 means Method P prefers three good or fair points when available.

Recommended:
Keep at 3.

-----------------------------------------------------

methodPMaxAccuracyM: 15,

What it does:
Sets the poorest GPS accuracy Method P normally accepts for its fitted calculation.

15 means readings worse than 15 metres accuracy are not accepted for the Method P fit.

This matches the Fair accuracy limit.

Recommended:
Keep at 15.

-----------------------------------------------------

methodPMaxResidualM: 18,

What it does:
Sets how far the GPS readings may deviate from Method P's fitted movement line before the fit is rejected.

18 means a maximum residual error of 18 metres.

In simple terms:
If the readings do not form a sensible enough movement pattern, Method P refuses the result.

Recommended:
Keep at 18 unless carefully tested.

-----------------------------------------------------

methodPMaxSpeedKmh: 160,

What it does:
Sets the highest believable vehicle speed accepted by Method P.

160 means 160 km/h.

A calculated result implying a higher speed is treated as unrealistic and rejected.

Recommended:
Keep at 160.

-----------------------------------------------------

methodPLockAfterFinalResult: true,

What it does:
Controls whether Method P stops changing once it has produced its final accepted result.

true  = the final Method P result is locked.
false = later GPS readings could continue changing it.

Recommended:
Keep this set to true.

=====================================================
BUTTON-MODE CONTROL SELECTION
=====================================================

buttonModeManualSelection: true,

What it does:
Controls how STCs are selected and advanced in Button mode.

true  = the user selects or advances the STC through the normal button/user sequence.
false = the app may rely more on automatic GPS-driven selection behaviour.

Important:
With this set to true, GPS does not visibly move the user through STCs in Button mode.

Recommended:
Keep this set to true.

-----------------------------------------------------

autoAdvanceHalfwayEnabled: false,

What it does:
Controls whether the app automatically advances toward the next STC when GPS judges that the vehicle is halfway there.

true  = halfway GPS position may trigger automatic advance.
false = halfway GPS position does not auto-advance the visible STC.

Recommended:
Keep this set to false.

This supports the rule that Button mode remains user-controlled and GPS stays in the background.

=====================================================
NORMAL RECOMMENDED SETTINGS
=====================================================

For normal rally use, the important settings should read:

    startGpsOnOpen: true,
    primaryMethod: 'M2',
    backupMethod: 'P',
    allowBackupIfBetterQuality: false,
    buttonModeShowGpsTrialDiagnostics: false,
    gpsDetectionGateM: 300,
    stcConfirmBuzzEnabled: true,
    stcConfirmBuzzMs: 200,
    buttonModeSubmitGpsEvidence: true,
    gpsModeTimesEditable: false,
    buttonModeManualSelection: true,
    autoAdvanceHalfwayEnabled: false

=====================================================
TO SHOW GPS TEST INFORMATION AGAIN
=====================================================

Find:

    buttonModeShowGpsTrialDiagnostics: false,

Change only false to true:

    buttonModeShowGpsTrialDiagnostics: true,

Save index.html and reload the app.

To return to the clean normal-use screen, change true back to false.

=====================================================
IMPORTANT EDITING RULES
=====================================================

1. Make a backup copy of index.html first.

2. Change only the value after the colon.

3. Do not change the setting name.

4. Do not remove quotation marks from:
       'M2'
       'P'

5. Do not remove commas at the end of setting lines.

6. Use lowercase:
       true
       false

7. Do not use:
       True
       False
       TRUE
       FALSE

8. After saving, fully close and reopen the PWA.

9. If the change does not appear, the browser may still be using a cached copy. Clear the app/site cache and reopen it.
